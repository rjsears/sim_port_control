Part 1: SimTech Guide

Day-to-day operations for simulator technicians

1.1 Getting Started

SimPortControl provides a simple interface for enabling and disabling internet access to flight simulators. This section covers how to log in and navigate the main dashboard.

Logging In

To access SimPortControl, open your web browser and navigate to the system URL provided by your administrator (e.g., https://simportcontrol.loft.aero).

Figure 1.1: The SimPortControl login page

  1. Enter your username in the first field
  2. Enter your password in the second field
  3. Click the Sign In button

Figure 1.2: Entering credentials

Note

If you don't have login credentials, contact your system administrator to create an account for you.

The Simulator Dashboard

After logging in, you'll see the main dashboard displaying all simulators you have access to. Each simulator is shown as a card with its icon and name.

Figure 1.3: Dashboard showing simulators with internet access disabled (gray icons)

When a simulator has active internet access, it displays with a colored icon, a green glow effect, and an "Internet Active" badge. If a single port is active, the remaining time is shown below the badge (e.g., "2h 30m"). When multiple ports are active, each port's timer is displayed with its interface name (e.g., "Gi1/0/7: 23h 59m").

Figure 1.4: Dashboard showing simulators with active internet access

Tip

The dashboard automatically refreshes to show current port status. You don't need to manually refresh the page.

1.2 Activating Internet Access

When a simulator needs internet connectivity (for updates, downloads, or remote support), follow these steps to enable access.

  1. Click on the simulator card from the dashboard
  2. On the simulator detail page, you'll see a list of ports showing the port number, switch name, and VLAN
  3. Click the green Enable button next to the port you want to activate

Figure 1.5: Simulator detail view with internet access disabled

A confirmation dialog will appear with a Duration dropdown that lets you choose how long the port should remain active.

Figure 1.6: Confirmation dialog for enabling internet access

  1. Select a duration from the dropdown (Default uses the port's configured timeout, or choose 1-24 hours)
  2. Click Enable to confirm activation

Once activated, the port status icon turns emerald green, the button changes to Disable, and a countdown timer displays the remaining time. You'll be automatically returned to the dashboard.

Figure 1.7: Simulator with active internet access showing countdown timer

Important

Internet access will automatically disconnect when the timer reaches zero. This is a safety feature to prevent accidental extended connectivity.

1.3 Deactivating Internet Access

When you're finished using internet connectivity on a simulator, you should manually deactivate it rather than waiting for the timeout.

  1. Navigate to the simulator detail page (click the simulator from the dashboard)
  2. Click the blue Disable button next to the active port
  3. A confirmation dialog will appear

Figure 1.8: Confirmation dialog for disabling internet access

  1. Click Disable to confirm deactivation

The port will immediately disconnect, the status icon turns blue, and the button changes back to Enable. You'll be automatically returned to the dashboard.

Figure 1.9: Simulator after internet access has been disabled

Best Practice

Always manually disable internet access when you're done using it. Don't rely solely on the automatic timeout.

1.4 Understanding Timeouts

SimPortControl uses automatic timeouts as a safety feature to ensure simulators don't remain connected to the internet indefinitely.

How Timeouts Work

  • Each port has a default timeout configured by the administrator (commonly 4 hours)
  • When you activate a port, a countdown timer begins
  • The timer is displayed on both the dashboard and simulator detail page
  • When the timer reaches zero, the port automatically disables

Figure 1.10: The countdown timer showing remaining connection time

Multiple Ports

Some simulators may have multiple network ports. Each port has its own independent timeout. When multiple ports are active, the dashboard displays each port's remaining time individually, stacked vertically on the simulator card (e.g., "Gi1/0/7: 23h 59m"). Times are sorted with the shortest time first.

Figure 1.11: A simulator with multiple ports showing different remaining times

Note

If you need extended access beyond the default timeout, contact an administrator who can use the Force Enable feature with a custom duration.

1.5 Changing Your Password

You can change your password at any time directly from the application.

  1. Look for the key icon button in the header (next to the Logout button)

Figure 1.12: Location of the Change Password button in the header

  1. Click the key icon to open the Change Password dialog
  2. Enter your current password
  3. Enter your new password (minimum 6 characters)
  4. Confirm your new password by entering it again
  5. Click Change Password

Figure 1.13: The Change Password dialog

If successful, you'll see a confirmation message.

Figure 1.14: Success notification after changing password

Security Reminder

Choose a strong password that you don't use on other systems. If you forget your password, contact an administrator to reset it.

Part 2: Administrator Guide

System configuration and management for administrators

2.1 Admin Dashboard Overview

Administrators have access to all SimTech features plus a comprehensive management dashboard. To access the admin area, click the Admin button (gear icon) in the header.

Figure 2.1: The Administrator Dashboard with all management sections

The admin dashboard provides quick access to:

Section Description
Users Manage user accounts and simulator assignments
Simulators Configure flight simulators and FTDs
Switches Manage Cisco switch connections
Port Assignments Configure port to simulator mappings
Discovered Ports Scan and view switch ports
Activity Logs View port control history
System SSL certificates and system info
Note

Admin users automatically have access to all simulators. You don't need to assign simulators to admin accounts.

2.2 User Management

The User Management section allows you to create, edit, and delete user accounts, as well as assign simulators to SimTech users.

Figure 2.2: User management table showing all users

Creating a New User

  1. Click the Add User button
  2. Enter a username (3-50 characters)
  3. Enter a password (minimum 8 characters)
  4. Select the user role

For SimTech users: Select which simulators they can access.

Figure 2.3: Adding a SimTech user with simulator assignments

For Admin users: No simulator selection needed—admins access all simulators.

Figure 2.4: Adding an Admin user (automatic access to all simulators)

Editing a User

Click the pencil icon next to a user to edit their details.

Figure 2.5: Editing an existing user

Tip

Leave the password field blank when editing to keep the current password unchanged.

Deleting a User

Click the trash icon to delete a user. A confirmation dialog will appear.

Figure 2.6: Delete user confirmation dialog

Warning

Deleting a user cannot be undone. The user's activity history will be preserved in the logs.

  5. Click Test Connection to verify

2.5 Port Assignments

Port assignments link switch ports to simulators, defining which port controls internet access for each device.

Figure 2.14: Port assignments table

Creating a Port Assignment

  1. Click Add Assignment
  2. Select the simulator
  3. Select the switch
  4. Enter the port identifier (e.g., "Gi0/1")
  5. Enter the VLAN number (default: 30)
  6. Set the default timeout in hours
  7. Click Create

Figure 2.15: Creating a new port assignment

Figure 2.16: Editing port assignment settings

Multiple Ports

A simulator can have multiple port assignments if it requires multiple network connections.

2.6 Port Discovery

The Port Discovery feature allows you to scan switches and discover available ports, making it easier to set up port assignments.

Figure 2.17: Port discovery view before scanning

  1. Select a switch from the dropdown
  2. Click the Scan button
  3. Wait for the scan to complete

Figure 2.18: Scanning in progress

After scanning, you'll see all discovered ports with their current status.

2.7 Activity Logs

The Activity Logs section provides a complete audit trail of all port control actions.

Figure 2.21: Activity logs showing port control history

Each log entry includes:

  • Timestamp: When the action occurred
  • User: Who performed the action
  • Action: Enable, disable, force enable, etc.
  • Simulator: Which simulator was affected
  • Port: The specific switch port
  • Reason: For admin force enables, the stated reason

Figure 2.22: Filtered activity logs

Tip

Use the activity logs to troubleshoot issues or review who accessed which simulators.

Clearing Logs

Admins can clear all activity logs by clicking the Clear Logs button in the header. A confirmation dialog will appear before deletion.

Warning

Clearing logs is permanent and cannot be undone. Consider exporting or backing up important audit data before clearing.

2.8 System & SSL Management

The System section displays overall system health and SSL certificate status.

Figure 2.23: System health overview

SSL Certificate Status

The SSL section shows your certificate details with color-coded expiration indicators:

Color Days Remaining Action Needed
Green More than 30 days None
Yellow 15-30 days Plan renewal
Red Less than 15 days Renew immediately

Figure 2.24: SSL certificate status display

Force Renewal

If needed, you can manually trigger certificate renewal:

  1. Click the Force Renew button
  2. In the dialog, optionally check Dry run to test without making changes
  3. Click Force Renew (or Run Dry Test if dry run is checked)
  4. Wait for renewal to complete (may take 60-90 seconds for DNS validation)
  5. Review the full certbot output displayed in the dialog
Dry Run Mode

The Dry run checkbox allows you to test the renewal process without actually requesting a new certificate. This is useful for verifying your DNS configuration and certbot setup without consuming Let's Encrypt rate limits. When dry run is enabled, the button changes to "Run Dry Test" and no certificate changes are made.

Figure 2.25: Force renewal confirmation dialog with dry run option

Figure 2.26: Successful SSL certificate renewal with certbot output

Automatic Renewal

The certbot container automatically checks for certificate renewal every 12 hours. Certificates are renewed when they have less than 30 days remaining. After successful renewal, nginx is automatically reloaded to use the new certificate. Manual renewal is rarely needed.

2.9 Force Enable (Admin Override)

Administrators can force-enable ports with custom timeout durations, including indefinite ("Always On") access.

Using Force Enable

There are two ways to force-enable a port:

Method 1: Using the Enable button with "Always On"

  1. Navigate to any simulator detail page
  2. Click the Enable button next to the port
  3. In the Duration dropdown, select Always On (until manually disabled)
  4. The Force Enable modal will open - enter a reason and click Enable Always-On

Method 2: Using the bolt icon

  1. Navigate to any simulator detail page
  2. Click the bolt icon (⚡) next to any port (visible only to admins)
  3. Enter a reason for the override (required)
  4. Click Enable Always-On to confirm

Figure 2.28: Force Enable modal requiring a reason

Duration options available to admins include extended timeouts (2 days, 3 days, 1 week) and the special Always On option.

Figure 2.29: Entering a reason for force enable

Always On Mode

When a port is enabled with "Always On", it displays a special badge and remains active until manually disabled. The main dashboard will show a "Locked On" indicator with a lock icon instead of a countdown timer.

Figure 2.30: Simulator with "Always On" badge and "Locked On" indicator

Disabling a Locked Port

To disable a force-enabled port, simply click the Disable button. This will disable the port on the switch and automatically remove the lock. There is no separate "Remove Lock" action needed.

Important

All force enable actions are logged with the admin's username and stated reason. Use this feature responsibly.

Appendix C: Cisco Switch Configuration

Setting up your Cisco switch for SimPortControl

Overview

SimPortControl communicates with Cisco switches via SSH to enable and disable switch ports. Before adding a switch to SimPortControl, you must configure SSH access and create an appropriate user account on the switch.

Important

The user account must have privilege level 15 to execute the necessary interface configuration commands.

Requirements

  • Cisco IOS switch (tested with 3560G, compatible with most IOS switches)
  • SSH v2 support enabled
  • Local user account with privilege level 15
  • Network connectivity between SimPortControl server and switch management interface

Step 1: Basic Switch Configuration

First, configure the switch hostname and domain name (required for RSA key generation):

configure terminal
hostname LOFT-SIM-SW01
ip domain-name loft.aero
end

Step 2: Generate RSA Keys

Generate RSA keys for SSH encryption. Use a minimum of 2048 bits for security:

configure terminal
crypto key generate rsa modulus 2048
end
Note

If RSA keys already exist, you may be prompted to replace them. Type "yes" to continue.

Step 3: Configure SSH Settings

Enable SSH version 2 and configure timeout and retry settings:

configure terminal
ip ssh version 2
ip ssh time-out 60
ip ssh authentication-retries 3
end

Step 4: Create SimPortControl User Account

Create a local user account with privilege level 15. This grants full configuration access required for port control:

configure terminal
username simportcontrol privilege 15 secret YourSecurePassword
end
Security Warning

Use a strong, unique password. This password will be stored encrypted in SimPortControl's database, but should still follow your organization's password policies.

Step 5: Configure VTY Lines for SSH

Configure the virtual terminal lines to accept only SSH connections with local authentication:

configure terminal
line vty 0 15
 login local
 transport input ssh
 exec-timeout 10 0
end

Step 6: Save Configuration

Save the configuration to persistent storage:

write memory

Step 7: Verify SSH Configuration

Verify that SSH is properly configured:

show ip ssh
show ssh

Expected output should show SSH version 2.0 enabled with your configured timeout values.

Complete Configuration Example

Here is the complete configuration sequence:

! Basic switch configuration
configure terminal
hostname LOFT-SIM-SW01
ip domain-name loft.aero

! Generate RSA keys for SSH
crypto key generate rsa modulus 2048

! Configure SSH settings
ip ssh version 2
ip ssh time-out 60
ip ssh authentication-retries 3

! Create user for SimPortControl
username simportcontrol privilege 15 secret YourSecurePassword

! Configure VTY lines for SSH only
line vty 0 15
 login local
 transport input ssh
 exec-timeout 10 0
end

! Save configuration
write memory

! Verify SSH is working
show ip ssh
show ssh

How SimPortControl Discovers Ports

SimPortControl uses the Port Discovery feature to scan switches and identify available interfaces. For a port to be discoverable, it must meet two criteria:

  1. Administratively down - The port must be in shutdown state
  2. Description = "Available" - The port description must be set to "Available" (case-insensitive)

Preparing Ports for Discovery

On your Cisco switch, configure unused ports that should be available for simulator assignment:

configure terminal
interface GigabitEthernet1/0/10
 description Available
 shutdown
end
write memory

Using Port Discovery

  1. Navigate to Admin → Discovered Ports
  2. Select a configured switch from the dropdown
  3. Click Scan Switch to initiate discovery
  4. SimPortControl connects via SSH and runs show interface description
  5. Only ports that are admin down with description "Available" are shown
  6. Click Assign next to an available port to assign it to a simulator
Note

Only GigabitEthernet (Gi) and FastEthernet (Fa) interfaces are discovered. VLAN interfaces, loopbacks, and other virtual interfaces are excluded.

Tip

When a port is assigned to a simulator, its description is automatically changed to SIMPORT:<simulator_name>. When released, it's changed back to "Available".

How Port Control Works

When a user enables or disables a port, SimPortControl executes the following commands via SSH:

Enabling a Port

configure terminal
interface GigabitEthernet1/0/7
 no shutdown
 switchport access vlan 30
end

Disabling a Port

configure terminal
interface GigabitEthernet1/0/7
 shutdown
end
