Part 1: SimTech Guide

Day-to-day operations for simulator technicians

1.1 Getting Started

SimPortControl provides a simple interface for enabling and disabling internet access to flight simulators. This section covers how to log in and navigate the main dashboard.

Logging In

To access SimPortControl, open your web browser and navigate to the system URL provided by your administrator (e.g., https://simportcontrol.yourdomain.com).

🖥️
Login Screen
01-login-screen.png

Figure 1.1: The SimPortControl login page

  1. Enter your username in the first field
  2. Enter your password in the second field
  3. Click the Sign In button
🖥️
Login Form Filled
02-login-filled.png

Figure 1.2: Entering credentials

Note

If you don't have login credentials, contact your system administrator to create an account for you.

The Simulator Dashboard

After logging in, you'll see the main dashboard displaying all simulators you have access to. Each simulator is shown as a card with its icon and name.

🖥️
Dashboard with Inactive Simulators
03-dashboard-inactive.png

Figure 1.3: Dashboard showing simulators with internet access disabled (gray icons)

When a simulator has active internet access, it displays with a colored icon, a green glow effect, and an "Internet Active" badge with the remaining time.

🖥️
Dashboard with Active Simulators
04-dashboard-active.png

Figure 1.4: Dashboard showing simulators with active internet access

Tip

The dashboard automatically refreshes to show current port status. You don't need to manually refresh the page.

1.2 Activating Internet Access

When a simulator needs internet connectivity (for updates, downloads, or remote support), follow these steps to enable access.

  1. Click on the simulator card from the dashboard
  2. On the simulator detail page, locate the network port icon (shown in red when disabled)
  3. Click the port icon to begin activation
🖥️
Simulator Detail - Port Disabled
05-sim-detail-inactive.png

Figure 1.5: Simulator detail view with internet access disabled

A confirmation dialog will appear asking you to confirm the activation. The dialog shows the default timeout duration for this port.

🖥️
Enable Confirmation Modal
06-enable-confirmation.png

Figure 1.6: Confirmation dialog for enabling internet access

  1. Review the timeout duration shown in the dialog
  2. Click Yes to confirm activation

Once activated, the port icon turns green and a countdown timer displays the remaining time until automatic disconnection.

🖥️
Simulator Detail - Port Enabled
07-sim-detail-active.png

Figure 1.7: Simulator with active internet access showing countdown timer

Important

Internet access will automatically disconnect when the timer reaches zero. This is a safety feature to prevent accidental extended connectivity.

1.3 Deactivating Internet Access

When you're finished using internet connectivity on a simulator, you should manually deactivate it rather than waiting for the timeout.

  1. Navigate to the simulator detail page (click the simulator from the dashboard)
  2. Click the green port icon (indicating active connection)
  3. A confirmation dialog will appear
🖥️
Disable Confirmation Modal
08-disable-confirmation.png

Figure 1.8: Confirmation dialog for disabling internet access

  1. Click Yes to confirm deactivation

The port will immediately disconnect, and the icon will return to red (disabled state).

🖥️
Simulator After Disabling
09-sim-detail-disabled.png

Figure 1.9: Simulator after internet access has been disabled

Best Practice

Always manually disable internet access when you're done using it. Don't rely solely on the automatic timeout.

1.4 Understanding Timeouts

SimPortControl uses automatic timeouts as a safety feature to ensure simulators don't remain connected to the internet indefinitely.

How Timeouts Work

  • Each port has a default timeout configured by the administrator (commonly 4 hours)
  • When you activate a port, a countdown timer begins
  • The timer is displayed on both the dashboard and simulator detail page
  • When the timer reaches zero, the port automatically disables
🖥️
Countdown Timer Close-up
10-countdown-timer.png

Figure 1.10: The countdown timer showing remaining connection time

Multiple Ports

Some simulators may have multiple network ports. Each port has its own independent timeout.

🖥️
Multiple Ports with Different Timers
11-multiple-ports-timers.png

Figure 1.11: A simulator with multiple ports showing different remaining times

Note

If you need extended access beyond the default timeout, contact an administrator who can use the Force Enable feature with a custom duration.

1.5 Changing Your Password

You can change your password at any time directly from the application.

  1. Look for the key icon button in the header (next to the Logout button)
🖥️
Password Button in Header
12-password-button.png

Figure 1.12: Location of the Change Password button in the header

  1. Click the key icon to open the Change Password dialog
  2. Enter your current password
  3. Enter your new password (minimum 6 characters)
  4. Confirm your new password by entering it again
  5. Click Change Password
🖥️
Change Password Modal
13-change-password-modal.png

Figure 1.13: The Change Password dialog

If successful, you'll see a confirmation message.

🖥️
Password Change Success
14-password-success.png

Figure 1.14: Success notification after changing password

Security Reminder

Choose a strong password that you don't use on other systems. If you forget your password, contact an administrator to reset it.

Part 2: Administrator Guide

System configuration and management for administrators

2.1 Admin Dashboard Overview

Administrators have access to all SimTech features plus a comprehensive management dashboard. To access the admin area, click the Admin button (gear icon) in the header.

🖥️
Admin Dashboard
15-admin-dashboard.png

Figure 2.1: The Administrator Dashboard with all management sections

The admin dashboard provides quick access to:

Section Description
Users Manage user accounts and simulator assignments
Simulators Configure flight simulators and FTDs
Switches Manage Cisco switch connections
Port Assignments Configure port to simulator mappings
Discovered Ports Scan and view switch ports
Activity Logs View port control history

Note

Admin users automatically have access to all simulators. You don't need to assign simulators to admin accounts.

2.5 Port Assignments

🖥️

Creating a Port Assignment

  1. Click Add Assignment
  2. Select the simulator
  3. Select the switch
  4. Enter the port identifier (e.g., "Gi0/1")
  5. Enter the VLAN number (default: 30)
  6. Set the default timeout in hours
  7. Click Create

2.9 Force Enable (Admin Override)

Administrators can force-enable ports with custom timeout durations, including indefinite ("Always On") access.

Using Force Enable

  1. Navigate to any simulator detail page
  2. Click the Admin Force Enable button (visible only to admins)
🖥️
Force Enable Button
41-force-enable-button.png

Figure 2.27: Admin Force Enable button on simulator detail page

  1. Select a timeout duration from the dropdown

Available options include:

  • Default (uses port's configured timeout)
  • 1 hour, 2 hours, 4 hours, 8 hours, 12 hours, 24 hours
  • 2 days, 3 days, 1 week
  • Always On (until manually disabled)
🖥️
Force Enable Modal
42-force-enable-modal.png

Figure 2.28: Force Enable modal with timeout options

  1. Enter a reason for the override (required)
  2. Click Force Enable to confirm
🖥️
Force Enable with Reason
43-force-enable-reason.png

Figure 2.29: Entering a reason for force enable

Always On Mode

When a port is enabled with "Always On", it displays a special badge and remains active until manually disabled.

🖥️
Always On Badge
44-always-on-badge.png

Figure 2.30: Simulator with "Always On" badge

Important

All force enable actions are logged with the admin's username and stated reason. Use this feature responsibly.
